## Overview

### What is the Contacts Module?

The Contacts Module is your **CRM (Customer Relationship Management) system** that stores and manages all people and organizations you interact with in your real estate business.

**Think of it like this**: Contacts are your business network - buyers, sellers, investors, vendors, partners. This module helps you maintain relationships and track interactions with everyone.

### What Can You Do?

âœ… **Store Contacts** - Save all people and companies you work with

âœ… **Search & Filter** - Find contacts instantly  
âœ… **Segment by Type** - Organize by buyer, seller, investor, etc.

### Method 3: Import from Lead

**Best for**: Converting leads to contacts.

**Steps**:
1. Open a **Lead** in Leads Module
2. Click **"Convert to Contact"**
3. Lead data auto-fills form
4. Review and complete any missing info
5. Click **"Create Contact"**
6. Lead and Contact are now linked!

### Method 4: Import from Transaction

**Best for**: Adding buyer/seller from completed transactions.

**Steps**:
1. Open a **Transaction** (Sell/Purchase Cycle)
2. In buyer/seller field, click **"Create New Contact"**
3. Transaction data pre-fills form
4. Complete additional details
5. Save contact
6. Auto-linked to transaction!

## Troubleshooting

### Common Issues

#### Issue: Duplicate Contacts

**Problem**: Same person appears twice.

**Solution**:
1. **Search first** before adding
2. **Merge duplicates** (Admin)
   - Keep most complete record
   - Transfer all interactions to primary
   - Delete duplicate
3. **Prevention**: Always search before adding

#### Issue: Can't Find Contact

**Problem**: Contact not appearing in search.

**Solutions**:
1. **Check filters**: Clear all filters
2. **Try partial name**: "Ahmed" instead of "Ahmed Khan"
3. **Search by phone/email**: Use other fields
4. **Check agent filter**: Admins see all, agents see their own

#### Issue: Wrong Contact Type

**Problem**: Contact has wrong type assigned.

**Solution**:
1. Open contact
2. Click **"Edit"**
3. Change **"Contact Type"**
4. Save changes
5. Type-specific fields update automatically

#### Issue: Missing Interactions

**Problem**: Interactions not showing in timeline.

**Solutions**:
1. **Check date filter**: Expand date range
2. **Check type filter**: Show all types
3. **Refresh page**: Force reload
4. **Admin review**: Escalate if persistent

## FAQs

### General Questions

**Q: What's the difference between a contact, lead, and deal?**  
A: 
- **Contact** = Person in your database (permanent record)
- **Lead** = Potential customer (active pursuit)
- **Deal** = Active transaction (offer/negotiation)

A contact can have multiple leads and deals over time.

**Q: Can one person be multiple types?**  
A: Yes and no. A contact has ONE primary type, but can have multiple roles:
- Primary Type: Buyer
- Also: Investor (via tags/notes)
- Also: Referral Source (via relationships)

**Q: How many contacts can I have?**  
A: Unlimited! But focus on quality relationships.

**Q: Can I delete contacts?**  
A: Admins can, but it's not recommended. Archive instead (keeps history).

### Data Management

**Q: How do I merge duplicate contacts?**  
A: Admin function only:
1. Identify duplicates
2. Select primary record (keep this)
3. Merge others into primary
4. All interactions/links transfer
5. Duplicates deleted

**Q: Can I import contacts from Excel?**  
A: Yes! Use Import feature:
- Prepare CSV file
- Map columns to fields
- Import in batches
- Review for duplicates

**Q: How do I export my contacts?**  
A: 
1. Go to Contacts
2. Apply filters (if needed)
3. Click **"Export"**
4. Choose format (CSV, Excel)
5. Download file

### Interactions

**Q: Do I need to log every interaction?**  
A: Yes! Every meaningful interaction should be logged for:
- Complete history
- Team visibility
- Performance tracking
- Legal protection

**Q: Can I edit past interactions?**  
A: Yes, but original timestamp is preserved (audit trail).

**Q: Can other agents see my contact interactions?**  
A: 
- Your contacts: You see all interactions
- Shared contacts: All assigned agents see interactions
- Admin: See all interactions (oversight)
