## How to Use the New System

### Generating a Report

1. **Navigate to Financials → Reports**
2. **Click a Report Card** (e.g., "Trial Balance" or "Changes in Equity")
3. **Click "Generate Report"** button
4. **Configure Parameters:**
   - Select date range (From/To dates)
   - Choose format (PDF/CSV/Excel)
   - Toggle comparison options if needed
5. **Click "Generate"**
6. **Report Opens Automatically!** ✨

### Viewing the Report

The report viewer opens with:
- **Professional formatting** - Clean table layout with proper headers
- **PKR currency formatting** - All amounts in Pakistani Rupees
- **Action buttons:**
  - 🖨️ Print - Print the report
  - 💾 Download - Export as selected format
  - 🔗 Share - Share with team (coming soon)
  - ❌ Close - Return to reports list

### Accessing Report History

**Method 1: From Template Card**
1. On any report template card, click the **"History"** button
2. See all instances of that specific report type
3. Click **"View"** on any report to open it
4. Click **🗑️** to delete from history

**Method 2: From Header**
1. Click **"Report History"** in the WorkspaceHeader (top-right dropdown)
2. See ALL reports across all types
3. Use search to find specific reports
4. Click any report to view it

### Managing Reports

**Search Reports:**
- Type in search box to filter by:
  - Report name
  - Date range
  - Generated by (user name)

**Delete Reports:**
- Click the 🗑️ trash icon on any report
- Confirm deletion
- Report removed from history

## Technical Implementation

### Data Flow

```
1. User clicks "Generate Report"
   ↓
2. GenerateReportModal collects parameters (dates, format, etc.)
   ↓
3. handleSaveReport() is called
   ↓
4. Appropriate accounting function is called:
   - generateTrialBalance() for Trial Balance
   - generateChangesInEquity() for Changes in Equity
   ↓
5. Report data is generated from actual ledger entries
   ↓
6. Full report object created with:
   - Metadata (dates, format, user, timestamp)
   - Data (actual accounting report object)
   ↓
7. Saved to localStorage
   ↓
8. ReportViewer opened immediately with report
   ↓
9. User can print, download, or close
   ↓
10. Report accessible anytime from History
```

### Data Structure

```typescript
interface ReportViewerData {
  id: string;                    // Unique ID (timestamp)
  templateId: string;            // e.g., 'trial-balance'
  templateName: string;          // e.g., 'Trial Balance'
  dateFrom: string;              // Start date (ISO format)
  dateTo: string;                // End date (ISO format)
  format: 'pdf' | 'csv' | 'excel';
  generatedAt: string;           // ISO timestamp
  generatedBy: string;           // User name
  data: TrialBalance | ChangesInEquity | any;  // ACTUAL REPORT DATA
}
```

### Storage

All reports stored in localStorage under key: `'generated_reports'`

### Integration with Accounting Library

The system integrates with `/lib/accounting.ts`:
- **generateTrialBalance()** - Creates trial balance from journal entries
- **generateChangesInEquity()** - Calculates equity changes from transactions

## Troubleshooting

### Report shows no data / empty accounts

**Cause:** No journal entries or transactions in the system yet

**Solution:** 
- Generate some journal entries first in General Ledger
- Or add some expenses/payments to have data
- Reports work with actual data from your accounting records

### History button shows empty

**Cause:** No reports generated yet for that template

**Solution:** Generate at least one report first

### Report viewer won't open

**Cause:** JavaScript error or missing data

**Solution:**
- Check browser console (F12) for errors
- Verify localStorage has 'generated_reports' key
- Try refreshing the page

### "This report type will be implemented soon" message

**Cause:** Report template other than Trial Balance or Changes in Equity

**Solution:** These will be implemented next:

- Cash Flow Statement
- Commission Report
